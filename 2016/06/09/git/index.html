GIT命令

1、github使用

  • 创建SSH key：ssh-keygen -t rsa -C "youremail@example.com"
  • 拷贝id_rsa.pub文件中的内容
  • 向github中添加ssh key即可

2、创建版本库

  • git init将这个目录变成git可管理的仓库
  • git add ./<filename>把文件添加到仓库(暂存区)
  • 每一次修改如果不add到暂存区，则不会加入到commit中
  • git commit -m "<state>"将文件提交到仓库 -m后输入本次提交的说明<将暂存区中的所有内容提交到当前分支(创建git版本库时会创建唯一一个master分支)>
  • git commit --allow-empty-message命令可以省略提交说明
  • git status显示仓库当前状态

3、版本管理

  • git log显示提交的历史记录(查看提交历史以便确定要回退到哪个版本)
  • git log --pretty=oneline只显示commit id
  • git reset --hard <version>版本回退命令，<version>有两种表示方法：
    • HEAD (git中HEAD表示当前版本，HEAD^表示上一个版本，HEAD^^表示上上个版本，往上100个版本用HEAD~100表示)
    • <commit id>指定回到过去或者未来的某一个版本
  • git reflog用来记录每一次命令(查看命令历史，以便确定要回到未来哪个版本)
  • git diff HEAD -- <filename>比较工作区中的文件和版本库中最新的版本的区别
  • git diff <filename>比较未commit的版本和上次修改的版本的区别(当文件add到暂存区中的时候此命令无效)
  • git checkout -- <filename>把文件在工作区中的修改全部撤销(让文件回到最近一次git add或者git commit后)
  • git reset HEAD <filename>可以将暂存区中的修改撤销掉，重新放回工作区(暂存区干净，工作区有修改，可以再使用git checkout -- <filename>命令撤销工作区中的修改)
    • rm <filename>将工作区中的文件删除
    • git rm <filename>将该文件从版本库中删除
    • git checkout -- <filename>撤销文件删除
  • 远程库版本rollback
    • git reset --hard <commit id> //将本地项目rollback
    • git push origin HEAD --force //将当前版本强制推送到远程服务器上

4、关联远程库以及分支管理

  • git remote add origin <gitaddress>在本地文件夹下运行此命令可以将本地文件夹与远程库关联起来
  • git push将本地库推送到远程库中
    • git push -u origin master把本地master分支推送到远程，并且将它们关联起来，第一次使用需要加-u参数，origin为远程库的名字，GIT的默认叫法，master也可以改为其他分支
    • git push origin master关联起来后就可以省略-u参数了
  • git checkout -b <name>创建分支并切换到该分支
    • git checkout <name>切换分支
    • git branch <name>创建分支
  • git branch查看当前分支
  • git merge <name>将分支上的新文件合并到master分支中(没有合并前分支上的修改在master分支上是不可见的)
    • --no--ff参数，不加此参数git会使用Fast forward模式(删除分之后会丢掉分支信息)，加上此参数后即为普通模式，可以看到合并历史
  • git branch -d <name>删除分支
    • git branch -D <name>还没有合并就要强行删除分支
  • 当分支中的内容和master分支中的内容都有修改后，使用git merge命令合并会提示冲突，这个时候使用git status命令找到冲突文件，打开文件修改冲突的地方，就可以git add .、git commit提交了
  • git log --graph查看分支合并图
  • git stash保存工作现场
  • git stash list查看保存的工作现场
  • 恢复工作现场
    • git status apply恢复 + git stash drop删除stash
    • git status pop恢复并删除stash
  • 如果有多个现场则可以使用git stash apply stash@{0}来选择恢复

5、多人协作

  • git remote查看远程库信息
    • git remote -v显示远程库详细信息
  • git clone默认只能看到master分支，如果需要在其他分支上开发，则必须创建远程origin的分支到本地git checkout -b <branch-name> origin/<branch-name>
  • 多人协作流程
    • git push origin <branch-name>推送分支
    • 推送分支失败时，先用git pull将分支先拉下来
    • 在本地解决冲突，就可以push上去了

6、标签管理

  • 创建标签
    • git tag <tag-name>
    • git tag <tag-name> <commit-id>指定commit打标签
    • git tag -a <tag-name> -m <state> <commit-id>，-a指定标签名，-m添加说明，-s通过私钥创建标签
  • git show <tag-name>查看标签信息
  • git tag -d <tag-name>删除标签创建的标签只存储在本地，不会自动推送到远程
  • git push origin <tag-name>推送指定标签到远程
  • git push origin --tags一次性推送所有未推送的标签
  • 删除远程标签
    • git tag -d <tag-name>先删除本地标签
    • git push origin :refs/tags/<tag-name>再删除远程标签

7、忽略特殊文件

  • 在根目录下创建.gitignore文件，配置该文件即可忽略特殊文件，最后将此文件提交到git即可
  • git add -f <filename>此命令可以强制添加提交文件

8、各种问题
